# Instructables Archive

Turn your Instructables profile into a searchable personal website with a full offline archive — hosted free on GitHub Pages.

**Live example:** [lonesoulsurfer.github.io](https://lonesoulsurfer.github.io)

---

## What you get

- A searchable, filterable website listing all your Instructables
- A complete offline archive — text, images, and attached PDFs for every project
- Lightbox image zoom, clickable parts-list links, and PDF download buttons
- Hosted free on GitHub Pages, your own custom URL

---

## One-click setup

You'll need:
- **Python 3.8+** — [python.org](https://python.org) (tick "Add to PATH" during install)
- **Git** — [git-scm.com](https://git-scm.com)
- **A GitHub account** — free at [github.com](https://github.com)

### Steps

1. Click the green **Code** button above → **Download ZIP** → extract it somewhere
2. **Windows:** double-click `run.bat`. **Mac/Linux:** double-click `run.sh` (or run `./run.sh` in Terminal)
3. Answer three questions: your GitHub username, Instructables username, and display name
4. The script will pause and tell you to create one empty repo on GitHub (takes ~30 seconds) — it gives you the exact name and a direct link
5. Press Enter and the rest happens automatically: clone, configure, scrape, archive, build, and publish

The archive step takes 1–2 hours for a large profile (smaller profiles are much faster), so start it and let it run.

Your site will be live at `https://YOUR_GITHUB_USERNAME.github.io` when it finishes.

## Manual setup (if you prefer step-by-step control)

<details>
<summary>Click to expand</summary>

### 1. Create your GitHub Pages repo

Create a repo named exactly `YOUR_GITHUB_USERNAME.github.io`, set it Public, don't add a README.

Enable Pages: **Settings → Pages → Source: Deploy from branch → main → / (root)**

### 2. Clone it and add the tool files

```bash
git clone https://github.com/YOUR_GITHUB_USERNAME/YOUR_GITHUB_USERNAME.github.io
cd YOUR_GITHUB_USERNAME.github.io
```

Copy all files from this repo's ZIP into that folder.

### 3. Install dependencies

```bash
pip install requests beautifulsoup4 playwright
python -m playwright install chromium
```

### 4. Configure

```bash
python setup.py
```

### 5. Scrape, archive, build

```bash
python scraper/scrape.py
python scraper/download_archive.py
python build_html.py
```

### 6. Push

```bash
git add .
git commit -m "initial archive"
git push
```

## Keeping it current

After publishing a new Instructable, double-click `update-site.bat` (Windows), or run:

```bash
python scraper/scrape.py
python scraper/download_archive.py --new-only
python build_html.py
git add .
git commit -m "add: new project"
git push
```

## File structure

```
your-repo/
├── run.py / run.bat / run.sh   ← one-click setup launcher
├── config.py                   ← your settings (auto-written by run.py)
├── setup.py                    ← manual config wizard (alternative to run.py)
├── build_html.py                ← builds HTML pages from archive
├── check_archive.py             ← checks archive completeness
├── update-site.bat              ← one-click updater for new projects
├── index.html                   ← main website (reads projects.json)
├── projects.json                ← generated by scraper
├── scraper/
│   ├── scrape.py                ← scrapes your profile → projects.json
│   └── download_archive.py      ← downloads everything as markdown + images
└── archive/
    └── instructables/
        └── project-name/
            ├── index.md
            ├── index.html
            ├── images/
            └── pdfs/
```

## Troubleshooting

**"Python is not recognised"** — reinstall Python and tick "Add to PATH", or restart your terminal after installing.

**Clone fails in run.py** — make sure you created the GitHub repo with the exact name shown, and that it's empty (no README/license added).

**Scraper seems stuck** — it's slow by design (polite delays between requests). A profile with 250 projects takes 1-2 hours.
