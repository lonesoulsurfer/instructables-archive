# Scraper Tools

Two scripts that keep the website and local archive up to date.

---

## First-time setup

```bash
pip install requests beautifulsoup4 markdownify
```

---

## scrape.py -- update the website

Run this whenever you publish a new Instructable. It scrapes your profile, pulls titles, URLs, thumbnails, and view counts, and writes `projects.json` which the website reads.

```bash
cd scraper
python3 scrape.py
```

Then commit and push:

```bash
git add ../projects.json
git commit -m "add: <title of new project>"
git push
```

Takes about 5 minutes for a full run. Safe to re-run any time -- existing data is cached so only new projects are fetched fresh.

---

## download_archive.py -- build the offline archive

Downloads every Instructable as markdown + images into `archive/instructables/`. This is your insurance policy if Instructables ever goes down.

**First run (full archive -- takes 1-2 hours):**

```bash
cd scraper
python3 download_archive.py
```

**After publishing a new Instructable (fast -- only downloads what's missing):**

```bash
python3 download_archive.py --new-only
```

**Archive a single specific project:**

```bash
python3 download_archive.py --url https://www.instructables.com/LED-Filament-Lamp/
```

## Git LFS setup (required before committing images)

The archive images will be several GB. GitHub requires Git LFS for files this large.

Run this once in the repo root before your first archive commit:

```bash
git lfs install
git lfs track "archive/**/*.jpg"
git lfs track "archive/**/*.jpeg"
git lfs track "archive/**/*.png"
git lfs track "archive/**/*.gif"
git lfs track "archive/**/*.webp"
git add .gitattributes
git commit -m "configure git lfs for archive images"
git push
```

GitHub gives you 1GB of LFS storage free. If your archive exceeds that you can either purchase more storage ($5/month for 50GB) or host the images separately and just keep the markdown in the repo.

## Repo structure

```
├── index.html                  ← website (reads projects.json)
├── projects.json               ← generated by scrape.py
├── scraper/
│   ├── scrape.py
│   ├── download_archive.py
│   └── README.md               ← this file
└── archive/
    ├── README.md               ← auto-generated index of all projects
    └── instructables/

        │   ├── index.md
        │   └── images/

        │   ├── index.md
        │   └── images/
        ...
```

## Workflow summary

| Task | Command |
|------|---------|
| Published a new Instructable | `python3 scrape.py` then push |
| Keep archive current | `python3 download_archive.py --new-only` then push |
| Full re-archive from scratch | `python3 download_archive.py` |
| Archive one specific project | `python3 download_archive.py --url <URL>` |
